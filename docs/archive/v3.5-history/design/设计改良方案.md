# 个人网页设计改良方案

> 基于 v3.5.0 现网实测 + 已锁定《Portfolio v3.5 Creative Direction》对照分析
> 日期：2026-07-18
> 原则：在已锁定的 **Developer Editorial** 设计语言内做"补足与强化"，不做视觉重启

---

## 0. 评估概览

### 0.1 当前状态：成熟但"克制过头"

经过对运行中站点的逐页审查（Home / Skills / ProjectDetail / Timeline / DecisionSection），结论是：**设计系统成熟度很高，但 Creative Direction 锁定的"Confident Mode"在落地时退回到了"Strict Mode"**。

具体表现：

| Creative Direction 锁定目标 | 实测落地状态 | 差距 |
|---|---|---|
| "克制但有态度" | 克制有余，态度不足 | 中 |
| "呼吸式节奏（紧-松-紧-松）" | 均匀列表，无呼吸感 | 大 |
| "字号对比 > 颜色对比" | 字号阶梯压缩（最大 48px） | 中 |
| "有一个让人记住的视觉符号" | `// eyebrow` 已落地，但全站一致性破损 | 中 |
| "1 主 + 1 辅叙事色" | Amber 主色独占，Slate Blue 几乎隐形 | 大 |
| "不是 README，像技术专栏" | 100% 文字 + 代码 + 图标，无视觉锚点 | 大 |

### 0.2 三大核心矛盾

1. **"反 README"目标 vs 纯文本现实**：Creative Direction §0 明确把"像 README"列为当前问题，但 v3.5 落地后整站依然是 0 张图片、0 个视觉锚点（`docs/assets/screenshots/` 里 19 张项目截图完全未使用）。克制变成了克制主义的自我束缚。
2. **Signature 系统定义了 6 个，落地了 4 个**：Mono Number Prefix（`// 01 ·`）在 §6.3 锁定，但代码里 eyebrow 全是 `// 精选项目` 无编号；About/Resume/Interview/AiPractice 四页连 `//` 前缀都丢了。Signature 是"让人 3 秒记住"的资产，一致性破损直接削弱识别度。
3. **Slate Blue 辅助色"叙事色"名存实亡**：§5.4 锁定 Amber（热忱）+ Slate Blue（冷静）双角色，但实测全站 95% 强调色是 Amber。Skills 分类色、Interview 分类色虽定义了 Slate Blue，但视觉权重太低，用户感知不到"学者冷静"的存在。

## 1. 改良方案（按优先级）

### P0 — 修复 Signature 一致性（低成本，高识别度收益）

**问题**：
- `src/pages/About.vue` eyebrow 是 `关于我`（无 `//`）
- `src/pages/Resume.vue` eyebrow 是 `简历`（无 `//`）
- `src/pages/Interview.vue` eyebrow 是 `面试准备`（无 `//`）
- `src/pages/AiPractice.vue` eyebrow 是 `AI 实践`（无 `//`）
- Home / Skills 用了 `//`，但全站没有 Mono Number Prefix

**改良**：

1. **统一 `// eyebrow` 全站覆盖** — 四个子页面 eyebrow 加回 `//` 前缀，与 Home/Skills 对齐。这是 Signature #1（§6.3 评级 ⭐⭐⭐⭐⭐）的最低一致性要求。
2. **落地 Mono Number Prefix** — 多 section 页面按 §6.3 #2 落地章节编号：
   - Home：`// 01 · 精选项目` / `// 02 · 技术成长` / `// 03 · 联系方式`
   - ProjectDetail：`// 01 · 项目背景` / `// 02 · 架构设计` / `// 03 · 关键决策` / `// 04 · 实现细节`
   - About：`// 01 · 工程定位` / `// 02 · 成长轨迹` / `// 03 · 关于本站`
3. **NavBar 当前页指示器统一** — 当前 `router-link-exact-active` 才显示下划线，`router-link-active`（如 `/projects/x` 命中"项目"）只变文字色。建议 active 状态也显示半透明下划线，强化"你在哪"的导航反馈。

**影响文件**：`About.vue` / `Resume.vue` / `Interview.vue` / `AiPractice.vue` / `Home.vue` / `ProjectDetail.vue` / `NavBar.vue`
**风险**：极低，纯文案 + 类名调整

### P1 — 建立"视觉锚点"，打破 README 感（中成本，高记忆点收益）

**问题**：整站 0 图片。Hero 的代码片段卡片是好的"代码证据"，但全站除了 ArchitectureDiagram SVG 外没有任何视觉变化。`docs/assets/screenshots/` 里有 19 张项目截图（江南出行 11 张、两地书 9 张、考试系统 8 张）完全闲置。

**改良**：

1. **ProjectDetail 引入"项目封面截图"** — 在 `ProjectHeader` 与 metrics 之间插入 1 张主截图（江南出行用 `jiangnan-passenger-home.png`，两地书用 `ldr-app-home.png`，考试系统用 `interactive-quiz-system-main-interface.png`）。
   - 处理：圆角 + 1px border + `shadow-md`，与卡片语言一致
   - 懒加载：`loading="lazy"`，首屏不阻塞 LCP
   - 暗色模式：`filter: brightness(0.9)` 微降，避免亮屏刺眼
2. **ProjectCard featured 卡片引入"截图缩略"** — Home 的 featured 卡片（江南出行）右侧/底部加一张小截图，与 normal 卡片拉开"封面化"差距（呼应 §7.2 "Featured 卡片封面化"决策，该决策锁定但未落地）。
3. **Hero 右侧 aside 第三块：架构缩略图** — 当前 Hero 右侧是 `代码片段 + stats`。可在 stats 下方加一张极小架构图缩略（复用 `jiangnan-architecture.svg`），建立"代码 → 数据 → 架构"的三段证据链。**注意**：此项需评估 LCP 影响，若威胁首屏性能则放弃，保留当前双块。
4. **About 页加入"工作环境/设备图"**（可选） — About 是人物页，纯文字略显单薄。可考虑 1 张克制的工作台/桌面照片（若无可不放，保持克制）。

**影响文件**：`ProjectHeader.vue` / `ProjectCard.vue` / `HeroSection.vue` / 内容 frontmatter（加 `cover` 字段）
**风险**：中。需评估图片体积（建议 WebP + 显式 width/height 防 CLS），不破坏性能规范（LCP < 2.5s）
**约束**：截图来自 `docs/assets/screenshots/`，**禁止含个人隐私/敏感信息**（隐私规则 §13）

### P2 — 强化 Editorial 排版尺度（低成本，高气质收益）

**问题**：Creative Direction §1.2 锁定"Editorial 三级层次"和"字号对比 > 颜色对比"，但实测：
- 最大字号 `--text-5xl` = 48px（Hero title / ProjectHeader title 桌面端）
- Section title 仅 `--text-4xl` = 36px
- 字体只有 Inter（UI 安全字体）+ JetBrains Mono，无 editorial display 字体

这导致"杂志气质"目标停留在 `//` 注释符号层面，排版本身没有 editorial 张力。

**改良**：

1. **引入 Display 字号 token** — 在 `tokens.css` 新增：
   ```
   --text-display: 3.5rem;   /* 56px — section title 桌面 */
   --text-hero: 4.5rem;      /* 72px — hero title 桌面 */
   ```
   仅用于 `@media (min-width: 1024px)` 的 Hero h1 和 Home/ProjectDetail 的 section title，移动端保持现有尺寸。建立"巨大标题 vs 紧凑正文"的 editorial 对比。
2. **Hero title 排版强化** — 当前 `客户端 · Serverless · 分布式系统 / 三类工程实践` 是两行等权。建议：
   - 第一行降为 secondary 色 + `--text-lg`，作为"领域声明"
   - 第二行 `三类工程实践` 升级为 `--text-hero` + Amber，作为"视觉锤"
   - 形成"小声明 → 大主张"的 editorial 节奏
3. **Section title 加 editorial 分隔** — 当前 section title 下方直接接内容，缺少"开章"仪式感。建议在 Home/Skills/Interview 的 section title 下方加一条 24px × 2px 的 Amber 短线（复用 Accent Line 语言，但水平方向，与 DecisionSection 的垂直 Accent Line 形成"水平开章 / 垂直强调"双重语法）。
4. **正文段落首行不做缩进，但拉大段落间距** — 当前 MarkdownContent 段落间距偏紧，editorial 专栏需要"段落 = 思考单元"的呼吸。建议 `--space-5` → `--space-6` 段落底距。

**影响文件**：`tokens.css` / `HeroSection.vue` / `Home.vue` / `ProjectHeader.vue` / `MarkdownContent.vue` / `global.css`
**风险**：低。需注意 72px 在 1024-1279px 区间的可读性，建议加 `@media (min-width: 1280px)` 才升到 72px

---

### P3 — Timeline 节奏与差异化（中成本，中收益）

**问题**：Timeline 是"呼吸式节奏"应该最强的 section，但实测 4 个 stage 视觉完全一致（除"主项目 1.2x"的微调）。"能力变化 / 学习重点 / 下一阶段 / highlights"四块结构在每个 stage 重复，扫描时找不到节奏断点。

**改良**：

1. **主项目 stage "卡片化"** — 当前主项目仅放大字号。建议主项目 stage 整体提升为 surface 卡片（`--color-surface` 背景 + border + `shadow-md`），与其它 stage 的"纯文字 + 左侧线"拉开"主角 vs 配角"的层次。这才是 §7.3 "主项目阶段放大 1.2x" 的视觉落地。
2. **"下一阶段"弱化为连接线** — 当前"下一阶段"与"学习重点"同等权重，但它是"过渡信息"。建议改为 stage 底部的 italic 小字 + → 箭头，视觉上连接到下一个 stage，建立"流水"节奏。
3. **highlights chip 按重要性分级** — 当前 4 个 chip 等权。建议第 1 个 chip（最核心成就）用 Amber border + Amber 文字，其余保持中性。建立"4 个里最该看哪个"的引导。
4. **upcoming stage 改"占位卡"** — 当前"下一步"stage 用 hollow dot + muted 处理，但结构与已完成 stage 完全一致，显得"未完成"。建议改为极简占位：只有日期 + 标题 + 一句话，去除"能力变化/学习重点"块，建立"未来 = 未知 = 留白"的语义。

**影响文件**：`TimelineSection.vue` / `timeline.md`（可能加 `highlights` 优先级标记）
**风险**：低，纯样式

### P4 — Hero 优化（低成本，中收益）

**问题**：
1. **平板（768-1023px）隐藏代码片段** — `HeroSection.vue` 在 768px 媒体查询里 `display: none` 了代码卡片。平板用户只看到文字 + stats，失去了 Hero 的"代码即证据"核心。平板占访客 10-15%，这不是可放弃的区间。
2. **Hero stats 与 featured 项目卡 metrics 重复** — Hero 显示 `3 / 218 / 97 / 236`，featured 卡片显示 `218 源文件 / 97 API / 236 测试 / 27 表`。`218/97/236` 三处重复，削弱了 stats 的"工程规模"叙事力。

**改良**：

1. **平板保留代码片段，降级呈现** — 768-1023px 不隐藏 snippet，而是：
   - 字号保持 `--text-xs`（12px）
   - snippet 高度限制 + 横向滚动
   - aside 改为单列：snippet 在上、stats 在下
   - 主区域与 aside 从 7fr/5fr 调整为 6fr/4fr 给 snippet 更多空间
2. **Hero stats 重构为"能力维度"而非"项目指标"** — stats 应传递"我是什么人"，而非"江南出行有多大"（那是 featured 卡片的活）。建议改为：
   - `3` 完整项目
   - `4` 周实习（企业级交付节奏）
   - `408` 考研方向（学术定位）
   - `0` 框架依赖（本站手写组件）
   
   或保留 `3 完整项目`，其余三项换成"能力维度"指标，避免与项目卡 metrics 撞车。

**影响文件**：`HeroSection.vue` / `Home.vue`
**风险**：低

---

### P5 — Slate Blue 辅助色真正落地（低成本，中收益）

**问题**：§5.4 锁定 Slate Blue 为"学者冷静"辅助色，但实测只在 Skills/Interview 分类色里出现，且色值（`#475569`）与中性 Slate 过于接近，用户感知不到"第二叙事色"。

**改良**：

1. **Timeline "学习重点" 块用 Slate Blue 标签** — 当前"能力变化"用 Amber（热忱，对的），"学习重点"也用 muted gray。建议"学习重点" label 用 Slate Blue，建立"Amber = 能力跃迁（结果）/ Slate Blue = 学习投入（过程）"的双色叙事。
2. **Interview 分类色加强可见度** — 当前 4 分类色点太小（6px）。建议色点放大到 10px + 分类标题文字也染色（不仅是点），让 4 分类在视觉上真正可区分。
3. **About Facts Panel 的"教育/考研"用 Slate Blue** — 教育、考研是"学者"属性，GitHub、方向是"工程师"属性。Facts Panel 的 label 色可按语义分：Amber（工程）/ Slate Blue（学术），让双色故事在人物页就建立。
4. **暗色模式 Slate Blue 再提亮一档** — 当前 dark 模式 `--color-accent-secondary: #94a3b8` 与 muted text `#94a3b8` 完全相同，导致辅助色"隐形"。建议 dark 模式提到 `#cbd5e1`（slate-300），与 muted 拉开对比。

**影响文件**：`tokens.css` / `TimelineSection.vue` / `InterviewCategory.vue` / `About.vue`
**风险**：低，需复检 WCAG AA 对比度

---

### P6 — Section 过渡与收尾（低成本，中收益）

**问题**：Creative Direction §4.2 Rhythm 5 "Pause（停顿）"锁定"section 间大留白"，但实测所有 section 都是 `padding: var(--space-20) 0` 均匀间距，无"停顿"感。Contact 收尾也偏弱。

**改良**：

1. **Section 间引入"章节断行"** — 在 Home 的 projects → timeline → contact 之间，加一条极淡的水平分隔线（`border-top: 1px solid var(--color-border)` + 上下 `--space-16`），建立"一章结束 / 下一章开始"的 editorial 断行。Timeline 已有 `border-block`，但 projects 和 contact 没有。
2. **Contact 收尾强化"signature"** — 当前 Contact 是 `// 联系方式` + 名字 + 2 个链接。建议：
   - 名字下方加一句"定位签名"（如 `后端 · 分布式 · 工程实践`），与 Footer 的 about-role 呼应但更精炼
   - 邮箱/GitHub 卡片右侧加一句"开放实习/校招机会"的状态行（若属实），用 Slate Blue 状态点 + 文字，让 Contact 不只是"联系方式列表"而是"邀请"
3. **Footer 收尾的"Built with"行加 git commit hash** — 当前是 `Built with Vue 3 + TypeScript + Vite`。建议加 `· commit abc1234`（短 hash），强化"工程师的网站"气质，呼应 §7.9 "最后更新来源：git commit 日期"决策。

**影响文件**：`Home.vue` / `ContactSection.vue` / `Footer.vue` / `vite.config.ts`（注入 commit hash）
**风险**：低

## 3. 实施路线图

按"识别度 → 气质 → 内容"顺序，分 4 个批次：

| 批次 | 内容 | 预估改动 | 收益 |
|---|---|---|---|
| **批次 1** | P0 Signature 一致性 + P6 Section 过渡 | 7 文件，纯样式/文案 | 全站识别度 + 节奏感立即提升 |
| **批次 2** | P4 Hero 优化 + P2 Editorial 排版尺度 | 5 文件，含 tokens 新增 | 首屏气质 + 杂志感 |
| **批次 3** | P3 Timeline 差异化 + P5 Slate Blue 落地 | 4 文件，纯样式 | 双色叙事 + 节奏呼吸 |
| **批次 4** | P1 视觉锚点（截图引入） | 3 文件 + frontmatter | 打破 README 感（需性能验证） |

每批次独立可发布，前批次不依赖后批次。批次 4 因涉及图片资源，需单独做性能验证（LCP / CLS / 体积），不合规则回退。

---

## 4. 验收标准

每批次完成后回归《Portfolio_v3.5_CREATIVE_DIRECTION.md》§9 验证：

- Signature 6 元素是否全站可见且一致
- Color Story 三角色（Amber / Slate Blue / Slate）是否都有可见权重
- Motion 5 节奏是否至少 3 种可感知（Inhale-Exhale / Cascade / Pause）
- design-quality.md "必备质量 8 项"是否满足 ≥4 项
- performance.md Core Web Vitals 是否全绿（LCP < 2.5s / CLS < 0.1）
