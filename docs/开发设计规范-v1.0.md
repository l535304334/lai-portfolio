## 1. 网站定位

### 1.1 网站名称

**中文定位：**「软件工程学生 · 技术成长档案」

**个人品牌化标题（`<title>` + Header Logo）：**

| 方案 | 标题 | 风格 |
|------|------|------|
| **推荐** | **`lai.dev`** | 简短、技术感、可记忆、可长期持有 |
| 备选 | `lai-se-portfolio` | 描述性强，偏学术 |
| 备选 | `赖同学的工程日志` | 亲切、有个人色彩 |

> 最终选择：
> - **浏览器 `<title>`**：`赖睿轩 | 软件工程学生 · 技术成长档案`
> - **Header Logo**：`lai.dev`
> - **首页 H1**：`从需求到系统的软件工程实践`
>
> 三层信息各司其职：浏览器标题含姓名利于搜索，Header Logo 简短利于记忆，首页 H1 一句话讲清楚网站内容。

### 1.2 一句话介绍

> 一个软件工程学生，三个完整项目，从需求分析到分布式调度的工程实践记录。

### 1.3 核心展示策略

**"不是告诉面试官我有多强，而是展示我怎么解决问题。"**

| 用户 | 核心诉求 | 网站策略 |
|------|---------|---------|
| **考研复试导师** | 学术潜力、独立思考、工程素养 | 每个项目突出「问题 → 方案对比 → 选择理由 → 验证结果」的思维链条 |
| **校招面试官** | 落地能力、代码质量、团队协作 | 突出可量化的工程指标（测试覆盖、API 数量、并发验证数据）和代码设计 |

每个项目页遵循统一叙事结构：

```
背景（为什么要做）
  → 挑战（难点在哪）
    → 方案（怎么解决，为什么选这个方案）
      → 实现（核心代码设计）
        → 验证（测试结果 / 压力数据）
          → 复盘（学到了什么，下次怎么做）
```

### 1.4 与普通简历网站的差异

| 普通网站 | 本网站 |
|---------|--------|
| 头像 + 自我介绍 + 项目列表 | 以项目为核心，每个项目是一个"微型技术案例" |
| "我会 Spring Boot" | "我在江南出行中用 Redisson 实现两阶段分布式锁，压测验证 100 订单 × 20 司机零重复分配" |
| 罗列技术栈 | 技术栈从项目中自然呈现，每个标签都有代码证据 |
| 一次性产品 | 长期更新的技术成长记录 |

## 2. UI 设计规范

### 2.1 整体视觉风格

**方向：Developer Academic — 克制、专业、有细节**

- 不追暗黑炫酷（那是模板网站的套路）
- 不走纯学术白底黑字（太枯燥，不像软件工程学生）
- 不走花哨动画（干扰内容阅读）

**核心原则：**

1. **内容优先**：排版服务于阅读，不抢内容
2. **克制配色**：大面积中性色 + 小面积强调色
3. **呼吸感**：大量留白，段落间距充裕
4. **代码感点缀**：等宽字体标注技术术语，代码块风格展示关键代码
5. **微交互提质感**：hover 状态、平滑过渡，但不喧宾夺主

  --color-border:        #e2e8f0    分割线/边框

  --color-text-secondary:#4a5568    辅助文字
  --color-text-muted:    #a0aec0    禁用/占位文字

  --color-accent-light:  #fef3c7    强调背景（标签、提示块）
  --color-accent-strong: #b45309    深强调（hover 状态）

功能色
  --color-success:       #059669    成功/通过

  --color-error:         #dc2626    错误/失败

  --color-java:          #b07219    Java 标签

  --color-ts:            #3178c6    TypeScript 标签
  --color-redis:         #dc382d    Redis 标签

#### 暗色模式

```

--color-surface:       #1e293b    卡片背景
--color-border:        #334155    分割线

**色彩使用规则：**
- 大面积只用 `bg` 和 `surface`
- 强调色只用于：链接、按钮、技术标签、关键数字/指标
- 一个页面最多 3 处使用强调色
- 代码块保持独立配色（不跟页面主色）

### 2.3 字体方案

```css
/* 正文字体 */
--font-body: 'Inter', 'Noto Sans SC', -apple-system, BlinkMacSystemFont, sans-serif;

/* 等宽字体 — 代码、技术术语、数字指标 */
--font-mono: 'JetBrains Mono', 'Fira Code', 'Consolas', monospace;

/* 字体大小 */
--text-xs:   0.75rem;    /* 12px — 标签、脚注 */
--text-sm:   0.875rem;   /* 14px — 辅助文字 */
--text-base: 1rem;       /* 16px — 正文 */
--text-lg:   1.125rem;   /* 18px — 段落导语 */
--text-xl:   1.25rem;    /* 20px — 小标题 */
--text-2xl:  1.5rem;     /* 24px — 章节标题 */
--text-3xl:  1.875rem;   /* 30px — 页面标题 */
--text-4xl:  2.25rem;    /* 36px — Hero 标题 */

/* 行高 */
--leading-normal: 1.6;   /* 正文 */
--leading-heading: 1.3;  /* 标题 */
--leading-code: 1.5;     /* 代码 */
```

**字体加载策略：**
- Inter 从 Google Fonts 加载（仅 `regular`、`500`、`600` 字重）
- JetBrains Mono 从 Google Fonts 加载（仅 `regular` 字重）
- 中文字体使用系统默认（不加载中文字体文件）
- `font-display: swap` 避免白屏

### 2.7 项目展示页风格

项目详情页是网站核心，设计需支持长时间阅读。

**布局：**
- 单栏居中（max-width: 800px），保留右侧空白
- 大段落之间用分隔线（非空白）明确分界
- 架构图使用 SVG/Mermaid 渲染（白底黑线）
- 代码块深色背景独立区块
- 关键指标用数字大卡展示

**代码展示：**
- 使用 Shiki 语法高亮
- 深色代码背景（不随页面主题切换）
- 行号显示
- 关键行高亮

### 2.8 暗色模式方案

**策略：跟随系统 + 手动切换**

```typescript
// 优先级：手动选择 > 系统偏好
const theme = useLocalStorage<'light' | 'dark' | 'system'>('theme', 'system')
const resolved = computed(() => {
  if (theme.value === 'system') {
    return window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light'
  }
  return theme.value
})
```

**实现方式：CSS 自定义属性切换**

```css
:root { /* 亮色变量 */ }
[data-theme="dark"] { /* 暗色变量覆盖 */ }
```

**切换按钮：** 导航栏右侧，太阳/月亮图标，三态循环：系统 → 亮色 → 暗色

## 4. 技术架构设计

### 4.1 技术选型

| 层级 | 选型 | 版本 | 理由 |
|------|------|------|------|
| 框架 | Vue 3 | 3.5+ | SFC + Composition API，已有经验 |
| 语言 | TypeScript | 5.5+ | strict 模式，已有经验 |
| 构建 | Vite | 6+ | 已有经验，HMR 极快 |
| 路由 | Vue Router | 4.x | 官方路由，createWebHistory + Vercel SPA 回退 |
| CSS | CSS Custom Properties + PostCSS | - | 设计令牌系统，无第三方 CSS 框架依赖 |
| UI 组件 | 手写组件 | - | 页面量少（~6 个），不引入 UI 库避免模板感 |
| Markdown | markdown-it | 14+ | 轻量，可扩展插件（代码高亮、锚点） |
| 代码高亮 | Shiki | 2+ | 构建时高亮，零运行时开销 |
| 图标 | Lucide Vue | - | 轻量、Tree-shakable |
| 字体 | Google Fonts (Inter + JetBrains Mono) | - | 只加载 3 个字重 |
| 分析 | Umami（可选） | - | 隐私友好，自托管 |
| 部署 | Vercel | - | 免费、自动 HTTPS、Git 集成 |

### 4.2 为什么不用的东西

| 不用 | 原因 |
|------|------|
| Element Plus / Naive UI | 页面量少，手写更快且不模板化 |
| Tailwind CSS / UnoCSS | 增加依赖，设计令牌 + scoped style 足够 |
| Nuxt / Next.js | 纯静态站点，不需要 SSR/SSG 框架 |
| Pinia | 无全局状态需求，路由 + props 足够 |
| GSAP / 动画库 | 动画不是重点，CSS transition 够用 |
| 后端 / 数据库 | 纯静态展示，Markdown 管理内容 |

### 4.3 路由方案

```typescript
// src/router/index.ts
import { createRouter, createWebHistory } from 'vue-router'

const routes = [
  { path: '/',                 name: 'home',      component: () => import('@/pages/Home.vue') },
  { path: '/projects/:slug',   name: 'project',   component: () => import('@/pages/ProjectDetail.vue') },
  { path: '/skills',           name: 'skills',    component: () => import('@/pages/Skills.vue') },
  { path: '/interview',        name: 'interview', component: () => import('@/pages/Interview.vue') },
  { path: '/resume',           name: 'resume',    component: () => import('@/pages/Resume.vue') },
  { path: '/about',            name: 'about',     component: () => import('@/pages/About.vue') },
  { path: '/:pathMatch(.*)*',  name: 'not-found', component: () => import('@/pages/NotFound.vue') },
]

const router = createRouter({
  history: createWebHistory(),  // 干净 URL，无 /#/
  routes,
  scrollBehavior(to, from, savedPosition) {
    if (savedPosition) return savedPosition
    return { top: 0 }
  },
})
```

**Vercel SPA 路由回退配置：**

```json
// vercel.json
{
  "rewrites": [
    { "source": "/(.*)", "destination": "/index.html" }
  ]
}
```

所有非静态文件请求回退到 `index.html`，由 Vue Router 接管。

- `createWebHistory` 产生干净 URL（`/projects/jiangnan-travel` 而非 `/#/projects/jiangnan-travel`）
- Vercel `rewrites` 确保刷新页面时不 404
- 所有页面懒加载
- 项目页通过 `slug` 参数动态渲染

### 4.4 项目内容数据流（构建时静态生成）

**策略：Vite 构建阶段将 Markdown 转换为静态 JSON/HTML，运行时零解析开销。**

```
开发时                          构建时（Vite Plugin）           运行时
───────                         ────────────────────           ──────
content/projects/               Vite 插件扫描 content/          Vue 组件
  jiangnan-travel.md  ──────→   读取 Frontmatter + 正文         直接 import
  love-letter.md                markdown-it 转 HTML                JSON
  exam-system.md                Shiki 语法高亮处理                   ↓
  interview/                    输出到虚拟模块                  ProjectDetail.vue
    jiangnan-travel.md          virtual:project-content          Interview.vue
    love-letter.md              每个 .md → { meta, html }
    exam-system.md
    general.md
```

**Vite 插件实现思路：**

```typescript
// vite.config.ts — 自定义插件
function projectContentPlugin() {
  const virtualModuleId = 'virtual:project-content'
  const resolvedVirtualModuleId = '\0' + virtualModuleId

  return {
    name: 'project-content',
    resolveId(id) {
      if (id === virtualModuleId) return resolvedVirtualModuleId
    },
    load(id) {
      if (id === resolvedVirtualModuleId) {
        // 1. 扫描 content/projects/*.md
        // 2. 解析每个文件的 frontmatter + markdown → HTML
        // 3. Shiki 在 Node 端处理代码块高亮
        // 4. 输出为 JS module: export const projects = [...]
        return generateProjectsModule()
      }
    },
  }
}
```

**Markdown 文件结构（Frontmatter）不变：**

```markdown
---
slug: jiangnan-travel
title: 江南出行智慧服务平台
subtitle: 面向江西的网约车调度平台
tags: [Java, Spring Boot, Redis, Vue 3, Docker]
date: 2026-07
featured: true
metrics:
  - label: 源文件
    value: 218
  - label: API 端点
    value: 97
  - label: 测试用例
    value: 155
github: https://github.com/l535304334/jiangnan-travel
---

## 项目背景
...
```

**运行时使用：**

```typescript
// ProjectDetail.vue
import { projects } from 'virtual:project-content'

const props = defineProps<{ slug: string }>()
const project = computed(() => projects.find(p => p.meta.slug === props.slug))
// project.meta → { title, tags, metrics, ... }
// project.html → 已高亮的 HTML 字符串，直接 v-html 渲染
```

**关键决策：**

- 不在客户端运行 markdown-it 和 Shiki（省 ~500KB+ JS 体积）
- 代码高亮在构建时完成（Node 端 Shiki 全速运行）
- 修改 Markdown 后 `npm run dev` HMR 自动重构建
- 面试内容同理：`content/interview/*.md` 构建时处理

## 附录：开发约定

### Git 提交规范

```
feat: 新功能
fix: 修复
refactor: 重构
docs: 文档
style: 样式
chore: 构建/工具
```

### 代码风格

- Vue SFC：`<script setup lang="ts">` → `<template>` → `<style scoped>`
- 组合式函数：`use` 前缀
- Props/Emits：TypeScript 泛型声明
- CSS：优先使用设计令牌变量，不硬编码颜色值
- 文件：一个文件一个组件，200 行以内

### 浏览器兼容

- Chrome 90+
- Firefox 90+
- Safari 15+
- 不支持 IE
