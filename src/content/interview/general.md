### Q1: 你在项目中遇到的最大技术挑战是什么？

**考察点：** 问题解决能力、技术深度、诚实度

**回答思路：**
1. 江南出行并发调度中的 ABA 竞态条件——tryLock 成功后未重读 DB，偶现重复分配
2. 问题定位：压测中偶现，加了详细日志才复现
3. 修复：加锁后立即 selectById 重查
4. 教训：分布式锁只保证互斥，不保证数据新鲜度；AI 生成的并发代码不能完全信任

### Q2: 你如何平衡 AI 辅助开发和独立思考？

**考察点：** AI 工具使用的成熟度、独立技术判断力

**回答思路：**
1. 建立了明确的人机分工：架构选择、安全设计、算法设计 → 人做决策；代码生成、文档、测试用例 → AI 辅助
2. 每行 AI 生成的代码都经过审查——ABA bug 就是审查时发现的
3. 江南出行项目约 60% 代码由 AI 生成，但 100% 的关键决策是人做的
4. AI 是工具不是替代品——用它加速重复劳动，把时间留给深度思考

**关键词：** 人机分工、代码审查、关键决策

### Q3: 你的技术学习路线是怎样的？

**考察点：** 学习能力、技术规划、自我驱动

**回答思路：**
1. Vanilla JS → React（Taro 小程序）→ Vue 3（江南出行前端）→ Spring Boot → Redis → Docker
2. 学习方式：项目驱动——不是先学完再写，而是项目需要什么学什么
3. 三个项目体现了技术栈的渐进扩展
4. 当前方向：分布式系统设计 + 考研计算机基础 + TypeScript 高级类型

**关键词：** 项目驱动学习、技术栈扩展、渐进式

### Q4: 为什么选择做这三个项目？

**考察点：** 项目选择判断力、需求洞察

**回答思路：**
1. 题库：帮助真实用户（医学考生）解决真实问题——单文件约束来自用户需求

4. 三个项目从简单到复杂，覆盖"独立工具 → 全栈产品 → 企业级系统"

**关键词：** 用户需求、AI 工程化、企业级、渐进复杂度
