## 项目背景

为医学考试（中医/临床）备考设计的纯浏览器端刷题工具。核心约束：**打开即用**——用户双击 HTML 文件即可开始刷题，无需安装、无需服务器、无需网络。独立设计并实现了全部 V9 功能，包括 CSS 设计令牌系统、SM-2 间隔复习算法、异步搜索索引构建、Chart.js 数据可视化仪表板、快照备份系统、自定义题目管理。经过 7 轮 RC 扫描修复。

## 解决的问题

医学考生需要一个便携、离线可用的刷题工具。通用刷题 App 需要安装且常含广告。这个单文件 HTML 方案做到了真正的"零配置启动"。

## 技术架构

```
单文件 HTML (14,563 行)
  ├── CSS (~4,800 行) — 设计令牌系统 (65+ 变量)
  ├── HTML (~1,100 行) — 模态框/导航/控制栏
  ├── 题目数据 (~5,950 行) — ALL_QUESTIONS 数组 (5,944 题)
  ├── 应用逻辑 (~2,600 行) — 状态管理/渲染/事件处理
  └── Chart.js (内联 205KB) — 数据可视化
         │
  localStorage (tiku_v8_* 前缀)
    ├── questions / wrong / done / fav
    ├── srs (SM-2 状态)
    ├── dailyHistory (90 天日志)
    └── snap_* (10 份快照)
```

**技术栈：** HTML5 · CSS3 (自定义属性) · Vanilla JavaScript (ES5+) · Chart.js v4.4.4 · localStorage

## 核心功能

- **多模式刷题**：正常/错题/收藏/复习模式，支持复合叠加
- **SM-2 间隔复习**：EF 因子调节、间隔调度、每日复习队列
- **模拟考试**：可配置来源/题量/时间，分页考试 + 倒计时 + 自动交卷
- **全局搜索**：异步分批索引构建 + 150ms 防抖 + `<mark>` 高亮 + `#ID` 精准定位
- **数据可视化**：4 个 Chart.js 图表（答题量折线、正确率趋势、科目雷达、错题环形）
- **快照系统**：最多 10 份备份，90% 体积压缩，自动淘汰
- **配额保护**：localStorage 配额感知 + 自动淘汰最旧快照 + 主动预警
- **深色模式**：CSS 变量切换，只覆盖 20 个变量实现全局主题
- **键盘快捷键**：17 个快捷键全覆盖，支持纯键盘操作

## 项目复盘

**做得好的：**
- YAGNI 决策：单文件架构是真实用户需求驱动的，不是偷懒
- SM-2 算法与错题系统的集成设计——在现有系统上叠加调度，最小侵入
- 配额管理的三层防御（压缩/淘汰/监控）
- 设计令牌系统让深色模式只改 20 个变量

**可以改进的：**
- 14,000 行单文件的编辑体验不够好（应考虑 V10 工程化拆分）
- 无自动化测试
- 搜索使用 indexOf 线性扫描（数据量 6000 题够用，但应讨论扩展方案）
- v1.0.0 已发布但仅单次 commit，未来迭代需建立持续集成流程

**学到的：**
- "简单"不等于"简陋"——单文件可以承载 SM-2、Chart.js、设计令牌等高级功能
- 约束驱动设计——"打开即用"的约束反而催生了创造性方案
- 配额管理是客户端存储的必修课
